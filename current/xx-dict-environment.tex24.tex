\begincom{lisp-implementation-type, lisp-implementation-version}\ftype{Function}

====Syntax====

**lisp-implementation-type ** //\noargs// → //description//

**lisp-implementation-version** //\noargs// → //description//

====Arguments and Values====

//description// - a //[[CL:Glossary:string]]// or **[[CL:Constant Variables:nil]]**.

====Description====

**[[CL:Functions:lisp-implementation-type]]** and **[[CL:Functions:lisp-implementation-version]]** identify the current implementation of \clisp.

**[[CL:Functions:lisp-implementation-type]]** returns a //[[CL:Glossary:string]]// that identifies the generic name of the particular \clisp\ implementation.

**[[CL:Functions:lisp-implementation-version]]** returns a //[[CL:Glossary:string]]// that identifies the version of the particular \clisp\ implementation.

If no appropriate and relevant result can be produced, **[[CL:Constant Variables:nil]]** is returned instead of a //[[CL:Glossary:string]]//.

====Examples====

<blockquote> (lisp-implementation-type) → "ACME Lisp" //or// → "Joe's Common Lisp" (lisp-implementation-version) → "1.3a" → "V2" //or// → "Release 17.3, ECO #6" </blockquote>
